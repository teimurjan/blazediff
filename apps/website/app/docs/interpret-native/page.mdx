# Interpret

import { Callout } from "nextra/components";

Structured image diff analysis. Takes two images, runs a pixel diff, then classifies the result into regions with content-aware change types, spatial positions, and severity scoring.

Built on the [`blazediff`](https://crates.io/crates/blazediff) Rust + SIMD engine — interpret is included as part of `@blazediff/core-native`.

## Installation

```sh npm2yarn
npm install @blazediff/core-native
```

Pre-built binaries are included for all major platforms — no compilation required:
- macOS ARM64 (Apple Silicon) & x64 (Intel)
- Linux ARM64 & x64
- Windows ARM64 & x64

## Features

- **Region detection** — connected-component labeling to find discrete change regions
- **Content-aware classification** — compares changed pixels against local background in both images to detect additions, deletions, shifts, color changes, and rendering noise
- **Spatial positions** — top-left, center, bottom-right, etc.
- **Severity scoring** — low / medium / high based on diff percentage
- **Shape analysis** — solid, contour frame, sparse, edge-dominated, mixed
- **Color delta & gradient stats** — per-region YIQ color difference and edge scoring
- **Structured summary** — grouped by change type for CI/review

## API Reference

### `interpret(image1Path, image2Path, options?)`

Runs a diff and returns structured analysis results.

#### Parameters

| Parameter    | Type               | Description                      |
| ------------ | ------------------ | -------------------------------- |
| `image1Path` | `string`           | Path to the first image          |
| `image2Path` | `string`           | Path to the second image         |
| `options`    | `InterpretOptions` | Comparison options (optional)    |

##### Options

| Option         | Type      | Default | Description                                              |
| -------------- | --------- | ------- | -------------------------------------------------------- |
| `threshold`    | `number`  | `0.1`   | Color difference threshold (0.0-1.0). Lower = more strict |
| `antialiasing` | `boolean` | `false` | Enable anti-aliasing detection to exclude AA pixels      |

#### Return Types

```typescript
interface InterpretResult {
  summary: string;
  totalRegions: number;
  regions: ChangeRegion[];
  severity: string;      // "Low" | "Medium" | "High"
  diffPercentage: number;
  diffCount: number;
  width: number;
  height: number;
}

interface ChangeRegion {
  bbox: BoundingBox;
  pixelCount: number;
  percentage: number;
  position: string;       // "TopLeft" | "Center" | "BottomRight" | ...
  shape: string;          // "SolidRegion" | "ContourFrame" | ...
  shapeStats: ShapeStats;
  changeType: string;     // "Addition" | "Deletion" | "ContentChange" | ...
  signals: ClassificationSignals;
  confidence: number;
  colorDelta: ColorDeltaStats;
  gradient: GradientStats;
}
```

<Callout type="info">
  **Severity thresholds:** Low = &lt;1% changed, Medium = 1-10%, High = &gt;10%.
</Callout>

## Usage

### Programmatic API

```typescript
import { interpret } from '@blazediff/core-native';

const result = await interpret('expected.png', 'actual.png');
console.log(result.summary);
// "Moderate visual change detected (1.87% of image, 12 regions).
//  Content changed: 4 regions (bottom, center).
//  Content added: 3 regions (right, bottom, bottom-left).
//  ..."

for (const region of result.regions) {
  console.log(`${region.position}: ${region.changeType} (${region.percentage.toFixed(2)}%)`);
}
```

### CLI Usage

Via `@blazediff/cli`:

```bash
blazediff-cli interpret expected.png actual.png
blazediff-cli interpret expected.png actual.png --output report.html
blazediff-cli interpret expected.png actual.png --threshold 0.05 --antialiasing
```

### CLI Options

```bash
blazediff-cli interpret [OPTIONS] <IMAGE1> <IMAGE2>

Arguments:
  <IMAGE1>  First image path (PNG, JPEG, or QOI)
  <IMAGE2>  Second image path (PNG, JPEG, or QOI)

Options:
  -t, --threshold <THRESHOLD>  Color difference threshold (0.0-1.0) [default: 0.1]
  -a, --antialiasing           Enable anti-aliasing detection
      --output-format <FORMAT> Output format (json or text) [default: json]
  -o, --output <PATH>          Write output to a file; `.html` generates an interactive report
  -h, --help                   Print help
  -V, --version                Print version
```

If `--output` ends with `.html`, the CLI writes an interactive review page with side-by-side images, clickable region rows, and keyboard navigation. In that mode it does not print JSON or text to stdout.

### Exit Codes

- `0` — Images are identical
- `1` — Images differ (JSON on stdout)
- `2` — Error

### Change Types

The classifier compares changed pixels against local background in both source images to determine what happened:

| Change Type | When |
|---|---|
| `Addition` | Content appeared — changed pixels blend with background in image 1 but stand out in image 2 |
| `Deletion` | Content disappeared — changed pixels stand out in image 1 but blend with background in image 2 |
| `Shift` | Content moved — a matched Addition + Deletion pair with similar luminance and pixel count |
| `ContentChange` | Content modified — significant change where both images have distinct content |
| `ColorChange` | Colors shifted — structure preserved (low edge change) |
| `RenderingNoise` | Artifacts — tiny or sparse regions with subtle color differences |
